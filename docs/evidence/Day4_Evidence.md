# Day4 证据卡：发布表单设计与数据提交

## 一、任务概述

Day4 的核心任务是在 Day3 Mock 数据的基础上，实现"校园轻集市"的发布功能。用户可以在发布页面选择业务类型、填写对应字段、通过表单校验后提交数据到 JSON Server，最终能够在对应列表页看到新发布的信息。

## 二、表单字段设计

### 字段设计思路

发布表单需要覆盖四类业务，每类业务的字段不完全相同。我采用了"类型切换表单"方案——用户先选择发布类型，再展示该类型对应的专属字段。所有类型共用标题、地点、描述三个通用字段，不同类型展示各自的专属字段：

| 发布类型 | 专属字段 | 设计理由 |
|----------|----------|----------|
| 二手交易 | 分类、价格、成色 | 二手交易的核心信息是"卖什么、多少钱、几成新"，这三个字段缺一不可 |
| 失物招领 | 类型(丢失/捡到)、物品名称、时间、联系方式 | 失物需要区分方向，物品名称和联系方式是找回的关键 |
| 拼单搭子 | 拼单类型、目标人数、截止时间 | 拼单的核心是"拼什么、差几人、什么时候截止" |
| 跑腿代办 | 任务类型、酬劳、取件地点、送达地点、截止时间 | 跑腿需要明确起点、终点、报酬和时间 |

### FormField 组件

我抽取了 `FormField.vue` 组件来统一表单项的展示结构——包含标签文字、必填标识（红色星号）和校验错误提示。这样每个表单项的结构一致，页面代码也更整洁。

## 三、表单校验规则

校验函数 `validate()` 在表单提交前执行，逐字段检查：

- **通用校验**：标题、地点、描述不能为空
- **二手交易**：分类必填、价格必须大于0、成色必填
- **失物招领**：物品名称和联系方式必填，"丢失"需填丢失时间，"捡到"需填捡到时间
- **拼单搭子**：拼单类型必填、目标人数需大于1、截止时间必填
- **跑腿代办**：任务类型必填、酬劳不能为负数、取件/送达地点必填、截止时间必填

校验不通过时，对应字段下方会显示红色错误提示文字，表单不会提交。

## 四、数据新增与页面联动

### 提交流程

1. 用户填写表单 → 点击"提交发布"
2. `validate()` 执行校验
3. 校验通过 → 调用对应的 API POST 方法（如 `addTrade()`）
4. JSON Server 接收数据，自动生成 id，写入 db.json
5. 页面显示成功反馈（绿色提示条）
6. 1 秒后自动跳转到对应列表页（如二手交易 → /trade）
7. 列表页在 `onMounted` 中重新请求数据，展示新发布的条目

### 新增 API 方法

在四个业务 API 文件中各补充了 POST 方法：

```
addTrade()      →  POST /trades
addLostFound()  →  POST /lostFounds
addGroupBuy()   →  POST /groupBuys
addErrand()     →  POST /errands
```

每个方法使用 `Omit<T, 'id'>` 类型，让 JSON Server 自动生成 id。

## 五、AI 协作记录

### AI 生成内容

AI 主要辅助生成了：
- `FormField.vue` 组件的模板结构和样式
- 表单校验函数 `validate()` 的基础框架（各字段的非空检查）
- POST 请求的 try/catch 错误处理结构
- 表单重置函数 `resetForm()` 的初始逻辑

### 人工调整内容

我主动判断并修改了以下内容：

**1. 校验规则细化**
AI 初始只做了非空判断（`if (!form.title)`），我补充了更具体的校验：
- 价格必须 `> 0` 而非仅仅 `!== ''`
- 目标人数必须 `> 1`（拼单至少需要2人）
- 酬劳必须 `>= 0`（允许免费跑腿）
- 失物招领根据类型分别校验 `lostTime` 或 `foundTime`

**2. 数据提交字段对齐**
AI 生成的提交数据与 db.json 字段不完全匹配。例如 AI 在拼单数据中遗漏了 `currentCount` 和 `publisher`，我补充了 `currentCount: 1`（发布者自己算1人）和 `publisher: '张同学'`（固定用户）。

**3. 交互反馈完善**
AI 只简单地设置了 `submitted` 布尔值。我改为使用 `feedback` 对象（含 `show`、`ok`、`msg` 三个属性），支持成功/失败两种状态的文字提示，并区分绿色和红色。

**4. 页面联动**
AI 没有实现提交后的页面跳转。我补充了 `router.push()` 在成功后的自动跳转，并使用 `routeMap` 根据发布类型跳转到对应列表页。

**5. 表单重置优化**
AI 的清空逻辑把所有字段都设为空字符串，但 `lostType` 默认值应该是"丢失"而非空字符串，我添加了特判处理。

**6. 类型切换时消除错误**
AI 没有处理切换类型后残留的上一个类型的校验错误。我添加了 `switchType()` 函数，在切换时清空所有错误提示。
